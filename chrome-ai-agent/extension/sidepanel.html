Permissions & Safety

Chrome AI Agent reads the active page so it can summarize, extract, and run requested non-sensitive browser actions.

All URLs lets the side panel work on normal websites. Debugger is used briefly to read a bounded accessibility tree, then detached. Storage keeps local settings, task drafts, and action history.

Page context is sent to the local backend and selected provider for your request. API keys and saved history stay local to this browser/backend setup.

The agent blocks password, OTP, card, payment, transfer, crypto, account deletion, and destructive actions. Non-sensitive requested actions run automatically after planning, and ordinary submit actions require explicit instruction.
